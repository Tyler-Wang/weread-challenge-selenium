## 背景
我准备增加一个通过PushShowDoc推送微信公众号信息

## 改动要求
1、新增一个与ENABLE_EMAIL同级别的全局变量ENABLE_PushShowDoc，用于控制是否启用PushShowDoc
2、新增一个全局变量PushShowDoc_Token，用于用户配置PushShowDoc的推送token
3、新增一个sendPushShowDocMsg方法，http请求，url:https://push.showdoc.com.cn/server/api/push/{token}，post方法，Content-Type:application/json，body内容样例如下：
'''
{
    //推送标题，必传
    "title":"",
    //消息内容。
    "content":"消息内容",
}
'''
响应样例如下:
'''
//成功样例
 {
   "error_code": 0,
   "error_message": "ok"
 }

//失败样例
{
   "error_code": 10103,
   "error_message": "url或token不正确"
 }
'''
4、sendPushShowDocMsg只需要推送subject和text，不需要图片
6、在notifyUser方法中，增加判断ENABLE_PushShowDoc决定是否调sendPushShowDocMsg
